# v1.2.0 — 2026-06-28

## New Features

### Customer Display — Dual-Screen with Real-Time Sync

A second Tauri window now serves as a customer-facing display. As the cashier operates the register, the display mirrors the live basket, totals, and payment state in real time.

- A **Monitor** button in the register header opens and closes the window; the button highlights when active.
- **Idle state:** full-screen dark welcome screen with store logo.
- **Active/Payment state:** persistent 70/30 split — scrollable item list on the left, payment panel on the right. Updates on every keystroke in tendered fields.
- **KHQR:** QR code renders live as soon as the packet is assembled.
- **Done state:** full-screen thank-you screen with change due; auto-reverts to idle after 5 seconds.
- Both windows share the same built frontend via a `?window=customer` query param — no separate entry point.

### Per-Item and Per-Transaction Discounts

Discounts can now be applied at two levels simultaneously.

**Per-item:**
- Inline controls on every cart row: type toggle `% | $/៛` and a value input.
- Percentage mode clamps 0–100; fixed mode deducts a flat amount in the item's native currency (KHR or USD).
- Discounted line total shows the original price struck-through.
- Discount clears automatically when the type is switched or the item is removed.

**Per-transaction:**
- Discount section in the payment ledger, between the subtotal card and settlement tabs.
- Fixed-amount input interprets value in the configured main currency and converts internally.
- KHQR always regenerates automatically to reflect the post-discount total.
- Customer display shows per-item badges, a struck-through subtotal, and the deduction line.

### Daily Sales Summary with Print Support

A new **Daily Summary** view is accessible from the main nav bar.

- Four stat cards: revenue, order count, gross profit, and margin.
- Payment-method breakdown with proportional fill bars.
- Top-5 products ranked by quantity sold.
- Prev/Next day navigation arrows.
- Native print via `getCurrentWebviewWindow().print()` in Tauri; falls back to `window.print()` in browser.
- New backend endpoint: `GET /api/summary/daily`

### Low Stock Alerts on Register

Cashiers are now proactively notified when products are running low (≤ 5 units by default, configurable via `?threshold=N`) without needing to open the Stock Manager.

- Amber pill badge on the "Manage Stock" nav button shows the low-stock item count.
- Dismissible banner below the scanner bar lists each low-stock item with a direct link to restock.
- Dismissed state resets automatically after visiting the Stock Manager.
- New backend endpoint: `GET /api/products/low-stock`

### Excel / CSV Import for Inventory

Products can now be bulk-imported from `.xlsx` or `.csv` files via a 3-step modal in the Stock Manager.

- **Step 1 — Upload:** file picker or drag-and-drop (native Tauri drag via a new `read_file_bytes` Rust command).
- **Step 2 — Column mapper:** auto-detects columns by fuzzy-matching header names; live preview table updates as columns are mapped; required fields (name, price) block submission until mapped.
- **Step 3 — Result summary:** reports inserted, updated, and skipped row counts.
- "Update existing by barcode" toggle controls upsert vs. insert-only behaviour.
- A `sample-import.csv` is included as a reference template.
- New backend endpoint: `POST /api/products/bulk`

### Bilingual Locale Expansion (Khmer / English)

All new surfaces are fully translated in both Khmer and English. Export modals in Stock Manager and Sales History were also fully localized, eliminating remaining hardcoded English strings.

- Customer display strings: welcome, tagline, totals, payment panel labels, thank-you screen.
- Discount controls, daily summary (including Khmer month names and DMY date order), low stock alert.
- Excel import modal: all steps, column labels, result strings.
- Stock Manager and Sales History export modals: column labels, scope options, date presets.

## Bug Fixes

- **Update checker:** Fixed a missing ref that caused the full update object from the initial `check()` call to not be stored, breaking the update flow on subsequent user actions.

## Internal / Infrastructure

- Added `core:webview:allow-create-webview-window` Tauri capability for the customer display window.
- Added `core:webview:allow-print` Tauri capability for daily summary print.
- Added `customer-display` to the capabilities windows list so the second window receives `core:default` and event permissions.
- New Rust command `read_file_bytes` to support native drag-and-drop file access in Tauri.
